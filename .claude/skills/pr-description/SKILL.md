# PR Description 생성 가이드

## 목표

`git log`와 `git diff`를 분석하여 아래 포맷의 PR description을 마크다운으로 생성합니다.
체크리스트 섹션은 포함하지 않습니다.

## 분석 단계 (작성 전 반드시 수행)

아래 명령어를 순서대로 실행하여 정보를 수집하세요:

1. **현재 브랜치 확인**: `git branch --show-current`
2. **커밋 목록 조회**: `git log main..HEAD --oneline`
3. **변경 파일 통계**: `git diff main..HEAD --stat`
4. **상세 diff 조회**: `git diff main..HEAD` (주요 파일 위주로 파악)
5. **오늘 날짜 확인**: `currentDate` 컨텍스트 또는 시스템 날짜로 ISO 주차 계산

**ISO 주차 계산 방법:**
- 오늘 날짜의 ISO 8601 주차 번호를 사용 (월요일 시작 기준)
- 버전 형식: `v연도.주차` (예: `v2026.11`)

## PR Description 포맷

아래 구조를 그대로 따르세요. 해당 없는 섹션(신규 기능, 리팩토링, 버그 수정, 신규 파일)은 생략합니다.

```markdown
# v20XX.XX — 한 줄 요약 제목

## 변경 사항

### ✨ 신규 기능

**기능 제목** (`파일 경로`)
- 항목 설명
- 항목 설명

### 🐛 버그 수정

**수정 내용** (`파일 경로`)
- 항목 설명

### ♻️ 리팩토링

**리팩토링 내용** (`파일 경로`)
- 항목 설명
- 항목 설명

### 🆕 신규 파일

| 파일 | 설명 |
|---|---|
| `경로/파일명` | 한 줄 설명 |
| `경로/파일명` | 한 줄 설명 |
```

## 작성 원칙

- **제목(v20XX.XX — ...)**: 변경의 핵심을 한 문장으로 요약
- **섹션 이모지 고정**: ✨ 신규 기능 / 🐛 버그 수정 / ♻️ 리팩토링 / 🆕 신규 파일
- **파일 경로**: 섹션 제목 옆에 백틱으로 명시 (여러 파일이면 대표 파일만)
- **설명 항목**: 사용자가 바로 이해할 수 있는 수준의 한국어 문장
- **신규 파일 테이블**: 새로 추가된 파일만 포함, 수정 파일은 제외
- **체크리스트 섹션 절대 포함 금지**
- **출력은 반드시 마크다운 코드블록(````markdown ... ````)으로 감싸서 출력** — 사용자가 그대로 복사·붙여넣기 할 수 있도록

## 출력 방식

description 본문 위에 아래 두 줄을 먼저 안내하세요:

```
**이번 주차**: v20XX.XX (XXXX년 X월 X일~X일, ISO XX주차)
**PR 제목 예시**: `feat: v20XX.XX — 한 줄 요약`
```

그 다음 줄부터 PR description 본문을 **마크다운 코드블록**으로 감싸서 출력합니다.
사용자가 코드블록 우측 상단의 복사 버튼으로 바로 복사할 수 있게 합니다.
